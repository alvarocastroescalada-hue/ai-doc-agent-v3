Qué hace
                                                                                                                                              
  - Procesa un documento (PDF/DOCX/etc.), extrae texto, lo divide en chunks y crea embeddings.                                                
  - Indexa los chunks en un vector store y recupera contexto relevante (incluye “golden stories” y memoria previa).                           
  - Llama a un LLM para: análisis de requisitos → extracción de historias → validación.                                                       
  - Normaliza historias (traceability + criterios de aceptación), aplica un quality gate determinista.                                        
  - Genera salidas: *.requirements.json, *.backlog.json, *.validation.json, *.retrieval.json y *.backlog.xlsx.                                
  - Ofrece CLI (npm run analyze -- <archivo>) y API (POST /analyze) con registro de runs.  


arrancar API REST: npm run api
invocar: curl -F "file=@documents/Vending.7.1.pdf" http://localhost:3000/analyze


npm run analyze -- documents/Vending.7.1.pdf

• Mejora continua en 3 frentes:
                                                                                                                                                                                                         
  - Learning guardrails                                                                                                                                                                                  
    Solo actualiza data/quality_patterns.json si el run supera umbrales mínimos de calidad/validación.                                                                                                   
  - Coverage por functionality                                                                                                                                                                           
    Calcula cobertura real de requirements.functionalities contra historias generadas y la añade al eval.json.                                                                                           
  - Hard constraints pre-save                                                                                                                                                                            
    Evalúa mínimos de historias, validation score, quality score y coverage funcional.                                                                                                                   
    Modo configurable: warn (default) o fail.                                                                                                                                                            

Config por .env:

  - HARD_CONSTRAINTS_MODE=warn o fail
  - MIN_QUALITY_SCORE=0.55
  - MIN_VALIDATION_SCORE=75
  - MIN_FUNCTIONALITY_COVERAGE=0.7

*.eval.json:

  - functionalityCoverage
  - hardConstraints (passed/violations/thresholds/values)
