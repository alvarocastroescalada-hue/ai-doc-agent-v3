AI Doc Agent

Flujo guiado: analizar, seleccionar run, corregir y reentrenar.

Guia de Uso

Ruta recomendada en 3 pasos

1. Analizar documento

Sube un PDF/DOCX/TXT/MD con requisitos claros.

  • Empieza con Usar golden stories activado.
  • Evita archivos escaneados sin OCR.
  • Revisa el runId generado.

2. Seleccionar run

Haz clic en la tabla de runs para cargar backlog y detalle.

  • Prioriza runs completed.
  • Carga historias con Cargar historias del run.
  • Usa el filtro de US para enfocarte.

3. Feedback y retraining

Corrige historias y envia feedback para mejorar patrones.

  • Ajusta role, want, soThat.
  • Asegura AC con formato DADO/CUANDO/ENTONCES.
  • Guarda borrador si no terminas en una sesion.
Como mejorar resultados de forma consistente

Checklist de calidad de entrada

  • Incluye actores explicitos (quien usa el sistema).
  • Separa funcionalidades atomicas en frases cortas.
  • AÃ±ade reglas de error, limites y validaciones.
  • Especifica integraciones externas y campos clave.

Checklist de correccion humana

  • 1 historia = 1 objetivo verificable.
  • Evita verbos vagos: gestionar, manejar, permitir.
  • Incluye al menos 1 criterio negativo por historia.
  • Asegura notas tecnicas utiles en notesHu.

Si el score sale bajo

  • Corrige y envia feedback con Aceptado = Si.
  • Repite 2-3 iteraciones en el mismo dominio.
  • Consulta metricas para validar mejora por run.
